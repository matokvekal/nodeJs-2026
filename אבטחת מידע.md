סיכום אבטחת Web מעודכן ל-2026

כולל דוגמאות קוד ו-Best Practices מודרניים

1. XSS – Cross Site Scripting
   מה זה?

הזרקת JavaScript זדוני לאפליקציה דרך קלט לא מסונן.

המטרה:

גניבת JWT / Cookies

התחזות למשתמש

שינוי UI

קריאת מידע רגיש

סוגי XSS
1️⃣ Reflected XSS

קוד מגיע דרך URL ומשתקף מיידית.

https://example.com?name=<script>alert(1)</script>
2️⃣ Stored XSS

קוד נשמר ב-DB ומורץ אצל משתמשים אחרים.

דוגמה:
צ׳אט שמכניס HTML לא מסונן.

3️⃣ DOM Based XSS

הקוד לא עובר דרך השרת — מתבצע בצד לקוח.

const name = location.hash;
document.getElementById("output").innerHTML = name; 2. הגנות נגד XSS (2026 Best Practice)
✔️ 1. לעולם לא להשתמש ב-innerHTML

במקום:

element.innerHTML = userInput;

להשתמש:

element.textContent = userInput;

או:

element.appendChild(document.createTextNode(userInput));
✔️ 2. Sanitization (במידה וחייבים HTML)

ספריה מומלצת 2026:

npm i dompurify
import DOMPurify from "dompurify";

const clean = DOMPurify.sanitize(dirtyHtml);
element.innerHTML = clean;
✔️ 3. Content Security Policy (חובה ב-2026)

ב-Node / Express:

import helmet from "helmet";

app.use(helmet.contentSecurityPolicy({
directives: {
defaultSrc: ["'self'"],
scriptSrc: ["'self'"],
objectSrc: ["'none'"],
upgradeInsecureRequests: []
}
}));

CSP היום הוא קו הגנה קריטי.

✔️ 4. Trusted Types (2026 מודרני)

בדפדפנים מודרניים:

<meta http-equiv="Content-Security-Policy"
      content="require-trusted-types-for 'script';">

מונע הזרקת JS גם אם מישהו משתמש ב-innerHTML.

3. CSRF – Cross Site Request Forgery
   מה זה?

גרימת בקשה לשרת בשם המשתמש כשהוא מחובר.

השרת סומך על ה-Cookie.

❌ טעויות נפוצות

שינוי DB דרך GET

ללא CSRF Token

SameSite=None ללא צורך

4. הגנות נגד CSRF (2026)
   ✔️ 1. לעולם לא לשנות מידע דרך GET
   GET /transfer-money?amount=500

אסור.

✔️ 2. SameSite Cookie

היום ברירת מחדל מומלצת:

Set-Cookie: token=abc;
HttpOnly;
Secure;
SameSite=Lax;

Strict למערכות רגישות.

✔️ 3. CSRF Token (Double Submit Pattern)

Server:

res.cookie("CSRF-TOKEN", token, { sameSite: "lax" });

Client:

fetch("/api/update", {
method: "POST",
headers: {
"X-CSRF-TOKEN": getCookie("CSRF-TOKEN"),
"Content-Type": "application/json"
},
body: JSON.stringify(data)
});
✔️ 4. Origin / Referer Validation
if (req.headers.origin !== "https://yourdomain.com") {
return res.status(403).end();
} 5. CORS – 2026 נכון

לא:

origin: "\*"

כן:

app.use(cors({
origin: ["https://app.yoursite.com"],
credentials: true
})); 6. אחסון JWT בדפדפן – מה נכון ב-2026?
❌ לא מומלץ:

LocalStorage

SessionStorage

חשוף ל-XSS.

✔️ מומלץ:

JWT בתוך Cookie:

HttpOnly
Secure
SameSite=Lax

גישה דרך Cookie בלבד.

7. DOM Injection Prevention

2026:

לא להשתמש ב-dangerouslySetInnerHTML (React)

להימנע מ-v-html (Vue)

להשתמש ב-template escaping מובנה

React כבר מבצע escape אוטומטי:

<div>{userInput}</div>
