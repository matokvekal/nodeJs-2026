CORS Demo — Client Page

This HTML file is opened directly from your filesystem (file://).
It tries to fetch data from localhost:4000 or localhost:4001.
Open the browser DevTools Console (F12) to see the CORS error clearly.

1. Server WITHOUT CORS (port 4000)

Start: npm start in 6cors folder

The browser blocks this because the server did not send
Access-Control-Allow-Origin headers.
Postman would work fine — only the browser enforces CORS.
Result will appear here...

2. Server WITH CORS (port 4001)

Start: npm run cors in 6cors folder

The server sends Access-Control-Allow-Origin headers.
The browser allows the response to be read.
Result will appear here...
