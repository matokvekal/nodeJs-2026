סוגי Events נפוצים — מדריך מלא לסטודנטים

⭐ Top 10 Events הנפוצים ביותר

error data end close connect disconnect message request exit uncaughtException
1

⚙️ Process Events

// קריטי לפרודקשן
process.on('exit', code => {
  console.log('Exiting with code:', code)
})

process.on('uncaughtException', err => {
  console.error('Uncaught!', err)
})

process.on('unhandledRejection', err => {
  console.error('Unhandled Promise!', err)
})
👉 קריטי לפרודקשן — מונע קריסות שקטות
2

📡 OS / Signals

// Graceful Shutdown
process.on('SIGINT', () => {
  console.log('Ctrl+C pressed — shutting down')
  server.close()
  process.exit(0)
})

process.on('SIGTERM', () => {
  console.log('PM2/Docker stopped us')
  server.close()
})
👉 Shutdown clean — חובה ב־PM2 / Docker
3

🌐 HTTP / Server Events

const server = http.createServer()

server.on('request', (req, res) => {
  res.end('Hello!')
})

server.on('connection', socket => {
  console.log('New TCP connection')
})

server.on('close', () => {
  console.log('Server closed')
})
👉 בסיס של כל שרת HTTP ב־Node
4

🌊 Streams (מאוד חשוב)

const stream = fs.createReadStream('file.txt')

stream.on('data', chunk => {
  console.log('Got chunk:', chunk.toString())
})

stream.on('end', () => {
  console.log('Done reading!')
})

stream.on('error', err => {
  console.error('Stream error:', err)
})

stream.on('close', () => {
  console.log('Stream closed')
})
👉 הכי נפוץ בעבודה אמיתית — קבצים, HTTP body, DB
5

📁 File System (fs)

import fs from 'fs'

// Watch for file changes
fs.watch('file.txt', (eventType, filename) => {
  console.log(`Event: ${eventType}`)
  console.log(`File: ${filename}`)
  // eventType = 'rename' | 'change'
})
👉 שימושי ל־hot reload, config watchers
6

🗄️ Database / Drivers

// MongoDB / Redis / RabbitMQ

client.on('connect', () => {
  console.log('DB connected!')
})

client.on('error', err => {
  console.error('DB error:', err)
})

client.on('disconnect', () => {
  console.log('DB disconnected!')
})
👉 כל DB driver = EventEmitter מתחת
7

✨ Custom Events (שלך)

import { EventEmitter } from 'events'

const emitter = new EventEmitter()

// Register listener
emitter.on('user:created', user => {
  console.log('New user:', user)
})

// Fire the event
emitter.emit('user:created', { id: 1, name: 'Dana' })
👉 Pattern חזק מאוד בארכיטקטורה — decoupling
8

🤝 Promise Events

// לא בדיוק EventEmitter אבל חשוב

fetch('/api/data')
  .then(res  => res.json())
  .then(data => console.log(data))
  .catch(err  => console.error(err))
  .finally(()  => console.log('Done'))

// async/await — אותו דבר
try {
  const data = await fetch('/api/data')
} catch (err) { console.error(err) }
👉 Event Loop based — חלק מאותה מערכת
9

⏱️ Timers

// Event Loop based
setTimeout(() => {
  console.log('Runs once after 1 sec')
}, 1000)

setInterval(() => {
  console.log('Runs every 2 sec')
}, 2000)

// Cancel
const id = setInterval(() => {}, 500)
clearInterval(id)
👉 גם Event Loop based — לא thread נפרד!

👷 Worker / Cluster

import { Worker } from 'worker_threads'

const worker = new Worker('./worker.js')

worker.on('message', msg => {
  console.log('Worker says:', msg)
})

worker.on('exit', code => {
  console.log('Worker exited:', code)

👉 CPU-heavy tasks — בלי לחסום Event Loop
11

🔌 WebSocket (מאוד נפוץ)

import { WebSocketServer } from 'ws'

const wss = new WebSocketServer({ port: 8080 })

wss.on('connection', ws => {
  ws.on('message', msg => {
    console.log('Received:', msg.toString())
    ws.send('Echo: ' + msg)
  })

  ws.on('close',  ()  => console.log('Client left'))
  ws.on('error',  err => console.error(err))
})
👉 Real-time apps — chat, games, dashboards
⚠️

🚨 Error Handling Pattern

// ❗ תמיד לטפל ב־error
// אם לא — Node יכול לקרוס!

emitter.on('error', err => {
  console.error('Handled error:', err)
})

stream.on('error', err => {
  console.error('Stream error:', err)
})

server.on('error', err => {
  console.error('Server error:', err)
})
❗ error event ללא listener = קריסה מיידית של Node!
⚡ Advanced — Event-Driven Architecture
import { EventEmitter } from 'events'
const bus = new EventEmitter()

// Multiple listeners on the SAME event → decoupling מלא
bus.on('order:created', order => sendEmail(order))
bus.on('order:created', order => updateInventory(order))
bus.on('order:created', order => logToAnalytics(order))

// One emit — three handlers fire automatically
bus.emit('order:created', { id: 42, item: 'laptop', price: 3500 })
👉 זה הבסיס של Microservices, Message Queues, ו־Event Sourcing — אותו רעיון, בקנה מידה גדול יותר
📌 סיכום — למה Events?
→

Node בנוי על Events — כמעט כל API פנימי הוא EventEmitter

→

כמעט כל דבר = async + event — network, files, timers, DB, OS

→

תמיד לטפל ב־error — זה ה־event הכי חשוב בכל emitter

→

Multiple listeners — כוח אמיתי: event אחד, פעולות מרובות, ללא coupling

→

Event Loop — כל Events, Promises, Timers רצים עליו
